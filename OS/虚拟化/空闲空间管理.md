## 1 空闲块确实是链表管理

```

每个 block 的头部通常包含：

```
+-----------+
| size      |
| next ptr  |
+-----------+
| payload   |
+-----------+
```

当 block 是空闲的：

```
size  = 块大小
next  = 指向下一个空闲块
```

所以：

✔ 你的理解基本成立。

---

## 2 已分配块只需要 size

一旦 block 被分配：

```
+-----------+
| size      |
+-----------+
| user data |
+-----------+
```

此时：

```
next 字段不再需要
```

所以 allocator 会把这个空间交给用户。

你写的：

> 对于分配出去的内存，next 为 magic

这个说法 **不严谨**。

真实情况通常是：

```
next 字段直接被覆盖为用户数据
```

因为用户的数据会写进去。

所以 allocator **并不依赖 next 字段**。

# 二、magic 的真正作用

你困惑的这个地方，其实是一个 **debug 技巧**。

有些 allocator 会这样：

```
next = 0xdeadbeef
```

这种值叫 **magic number**。

作用只有一个：

## 检测非法操作

比如：

### double free

```
free(ptr)
free(ptr)   // 错误
```

allocator 检查：

```
if(next != MAGIC)
   error
```

---

### use after free

```
ptr = malloc()
free(ptr)
ptr->x = 10
```

如果 allocator 在 free 时：

```
memset(ptr, 0xCC)
```

程序会更容易崩溃。

---

所以总结：

| 字段 | 空闲块   | 已分配块       |
| ---- | -------- | -------------- |
| size | 有效     | 有效           |
| next | 链表指针 | 无意义 / debug |

magic **不是核心机制**，只是调试工具。

### allocator 里的 implicit free list

| size  | alloc  |
+-------+--------+
| data            |
+-----------------+

遍历方法是：

next_block = current + size
```

也就是说：

```
地址 + size = 下一个 block

```
链表关系没有显式存储

```
struct node{
    size
    next
}
```

这种叫：

```
explicit free list
```

所以：

| 概念          | 含义                   |
| ------------- | ---------------------- |
| implicit list | 通过 size 计算下一个块 |
| explicit list | next 指针链表          |

你其实描述的是第二种。

# 八、思考题 5：slab allocator

这个是操作系统里非常重要的机制。

在
Linux kernel
中，大量对象大小是固定的：

```
task_struct
inode
file
dentry
```

如果每次都：

```
malloc
free
```

成本很高。

于是引入 **slab allocator**。

---

## slab 的思想

假设我们需要大量：

```
task_struct
```

创建一个 cache：

```
task_struct cache
```

然后分配一个 slab：

```
+----------------------+
| obj | obj | obj | obj|
+----------------------+
```

每个 obj 大小相同。

---

### slab 的好处

1️⃣ 分配极快

```
pop freelist
```

2️⃣ 无碎片

对象大小固定。

3️⃣ cache friendly

对象连续。

4️⃣ 可以复用对象

避免频繁初始化。

---

### slab allocator 结构

```
cache
 ├── slab (partial)
 ├── slab (full)
 └── slab (empty)
```
