---

## 1) 三级知识树（按依赖顺序排列）

### 第一层：基础认知
- **1.1 搜索引擎基本概念**
  - 倒排索引、分词、相关性评分（TF-IDF / BM25）
- **1.2 Elasticsearch 核心术语**
  - 索引（Index）、文档（Document）、类型（Type，7.x 后废弃）、字段（Field）、映射（Mapping）
- **1.3 Elasticsearch 架构概览**
  - 节点（Node）、集群（Cluster）、分片（Shard）、副本（Replica）

### 第二层：核心原理与机制
- **2.1 数据写入与存储机制**
  - 文档如何被索引（近实时搜索、refresh、flush、translog）
- **2.2 查询与检索原理**
  - 查询 DSL（Query DSL）、布尔查询、全文检索 vs 精确匹配、打分机制
- **2.3 分布式架构原理**
  - 主分片与副本分片分配、故障恢复、一致性模型（quorum 写入）
- **2.4 映射与分析器（Analyzer）**
  - 动态映射 vs 显式映射、内置分析器（standard, keyword, whitespace 等）、自定义分析器

### 第三层：Java 集成与工程实践
- **3.1 官方 Java 客户端选型**
  - High Level REST Client（已弃用） vs Java API Client（8.x 推荐）
- **3.2 Java 中操作 ES**
  - 创建索引、增删改查文档、构建复杂查询、聚合（Aggregation）
- **3.3 性能与可靠性优化**
  - 批量操作（Bulk）、连接池配置、重试机制、超时控制
- **3.4 与 Spring Boot 集成**
  - 使用 spring-data-elasticsearch 或直接集成官方客户端
- **3.5 监控与调试**
  - 使用 _cat API、慢日志、profile 查询、explain API

## 2) 每一块的“掌握标准”

| 模块                           | 掌握标准                                                     |
| ------------------------------ | ------------------------------------------------------------ |
| **1.1 搜索引擎基本概念**       | 能解释倒排索引结构，并说明它为何适合全文检索；能区分 TF-IDF 与 BM25 的差异。 |
| **1.2 Elasticsearch 核心术语** | 能准确说出 Index/Document/Mapping/Shard 的含义，并在类比关系型数据库时不出错。 |
| **1.3 Elasticsearch 架构概览** | 能画出单节点/多节点集群结构图，说明主分片和副本的作用及数量设置原则。 |
| **2.1 数据写入与存储机制**     | 能描述从写入请求到可被搜索的全过程（含 refresh/flush/translog），并解释“近实时”含义。 |
| **2.2 查询与检索原理**         | 能手写 Query DSL 实现组合查询（must/should/filter），并解释 filter 为何不打分。 |
| **2.3 分布式架构原理**         | 能说明写入时如何保证多数分片成功才返回成功，以及节点宕机后数据如何恢复。 |
| **2.4 映射与分析器**           | 能为中文、英文、ID 类字段分别设计合适的 mapping 和 analyzer，并解释原因。 |
| **3.1 官方 Java 客户端选型**   | 能说明为何 High Level REST Client 已弃用，并能初始化 Java API Client（8.x）。 |
| **3.2 Java 中操作 ES**         | 能用 Java 代码完成 CRUD、Bool 查询、terms 聚合等常见操作，且处理异常。 |
| **3.3 性能与可靠性优化**       | 能使用 Bulk API 批量导入 10 万条数据，并配置合理的超时与重试策略。 |
| **3.4 与 Spring Boot 集成**    | 能在 Spring Boot 项目中通过配置自动注入 ES 客户端，并封装 Repository 层。 |
| **3.5 监控与调试**             | 能使用 _explain API 分析某条文档为何被命中/未命中，并解读 profile 结果。 |
