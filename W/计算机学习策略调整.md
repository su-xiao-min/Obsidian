# 一、三本书到底在讲什么层级

计算机系统其实可以粗暴地分成三层：

```
应用程序
↑
操作系统
↑
硬件
```

而你提到的三本书正好卡在这三个层级之间。

| 书                     | 研究对象              | 层级        |
| ---------------------- | --------------------- | ----------- |
| 《计算机组成原理》     | CPU / 内存 / 指令执行 | 硬件        |
| 《深入理解计算机系统》 | 程序如何在机器上运行  | 硬件 ↔ 软件 |
| 《操作系统导论》       | OS 如何管理资源       | 系统软件    |

换句话说：

```
程序员写的代码
        ↓
《深入理解计算机系统》
        ↓
操作系统
        ↓
《操作系统导论》
        ↓
硬件
        ↓
《计算机组成原理》
```

所以三本书的关系其实是：

```
组成原理  →  CSAPP  →  操作系统
(硬件)       (桥梁)      (系统)
```

但现实世界的学习顺序通常是：

```
CSAPP → 操作系统 → 组成原理
```

原因很简单：

**组成原理太硬核了。**

如果先啃那个，你可能会开始怀疑人生。

# 二、《CSAPP》这本书为什么这么怪

你现在看这本书头大是正常的。

深入理解计算机系统
原名 **Computer Systems: A Programmer's Perspective**

这本书的核心目标是：

> **让程序员理解程序在机器上发生了什么。**

所以它会讲很多奇怪的东西：

| 内容     | 目的         |
| -------- | ------------ |
| 汇编     | 理解编译器   |
| 内存布局 | 理解指针     |
| 缓存     | 理解性能     |
| 虚拟内存 | 理解操作系统 |
| 链接     | 理解程序加载 |

它不是一本：

- 操作系统书
- 计算机组成书
- 编译原理书

而是 **三者的混合体**。

所以读的时候会有一种感觉：

> “这书怎么什么都讲一点，但又不讲透。”

没错，它就是这样设计的。

# 三、《操作系统导论》在干什么

Operating Systems: Three Easy Pieces
中文叫《操作系统导论》。

这本书非常友好，它核心讲三件事：

```
Virtualization
Concurrency
Persistence
```

翻译成人话：

1️⃣ **虚拟化**

- 进程
- 虚拟内存
- CPU 调度

2️⃣ **并发**

- 线程
- 锁
- 同步

3️⃣ **持久化**

- 文件系统
- 磁盘

这本书其实是**现代操作系统最好的入门书之一**。

它比很多国内教材强太多。

# 四、《计算机组成原理》是另一种世界

计算机组成原理

这本书关注：

```
CPU
ALU
流水线
缓存
总线
指令系统
```

这基本是 **硬件工程师的世界**。

对于后端工程师来说：

你只需要理解几个东西：

- CPU cache
- 内存层级
- 指令执行
- cache miss

剩下的：

比如

- 微指令
- 控制器设计
- 门电路

面试几乎不会问。

# 七、关于 C 语言的问题

你发现一个现象：

> 《C Primer Plus》里的 C
> 和操作系统里的 C
> 完全不是一个东西。

观察力不错。

原因很简单：

```
C Primer Plus
↓
教语法

操作系统代码
↓
用 C 操控内存
```

操作系统里的 C 会大量使用：

```
指针
结构体
宏
位运算
```

看起来就像：

```
*(p + 8) = (*(p + 8) & ~0x7) | perm;
```

第一次看都会怀疑世界。

不用焦虑。

**你不需要会写操作系统。**

你只需要：

- 看得懂
- 理解思想

# 八、Windows 跑不了的问题

这点你观察得也很准确。

很多系统代码默认是：

```
Linux
POSIX
```

而 Windows：

```
WinAPI
NT kernel
```

接口完全不同。

所以很多书里的代码只能在 Linux 跑。

最简单解决方案：

**直接装 Linux。**

要么：

```
WSL2
```

要么：

```
Ubuntu 虚拟机
```

别在 Windows 上折腾系统编程。

那是自虐行为。
