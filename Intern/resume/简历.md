项目名称：珞珈派

**技术栈**：Spring Boot、Spring Security、MyBatis、MySQL、Redis、Elasticsearch、RabbitMQ、Docker

**项目描述**：珞珈派是一个前后端分离的社区项目，包括 PC 前端和管理后台，面相高校师生群体，旨在实现信息的高效发布与内容共享。

**责任描述**：作为核心研发人员，主要负责用户权限管理、消息通知和文章模块开发工作。

核心技术：

1. 采用 JWT 双令牌机制，通过 Access Token 处理业务请求，通过 Refresh Token 实现用户无感的令牌刷新。
2. 将用户评论、点赞、收藏、系统消息发送到 RabbitMQ，实现消息的异步解耦，提供系统效率和服务稳定性
3. 通过 Redis 实现计数统计与用户活跃度排行，并通过先写 MySQL 再删除 Redis 的方案保证高并发场景下的缓存一致性。
4. 采用 HandleExceptionResolver 的全局异常处理策略，提高了代码的健壮性和可维护性。
5. 采用 ResponseEntity 的统一回应，保证前端得到的状态码始终有意义。
6. 采用 RESTful 的接口风格，声明有意义的资源路径。

项目名称：珞珈图谱

技术栈：Spring Boot、MySQL、DeepSeek-OCR、Elasticsearch、Redis、Docker、OSS 服务。

项目描述：珞珈图谱是一个基于 DeepSeek-OCR & Elasticsearch 的图片检索平台，允许用户上传文物图片构建专属的图片领域空间。也允许用户上传技术文档指定关联的图片。它结合了大语言模型与基于大语言模型的 OCR 技术，允许使用自然语言在空间内检索图片和相关文档。

核心职责：

1. 利用 Spring Security+JWT 实现基于组织标签的 RBAC 的多级权限系统，通过用户角色、组织归属和文件属性的权限过滤，实现精细化的文档访问控制，确保敏感数据安全。
2. 采用 OSS 预签名技术，前端申请上传图片，后端返回临时上传地址，图片直接上传到 OSS 服务器，提高系统效率。
3. 本地部署 DeepSeek-OCR 模型，同时设计回调机制，要求传递图片 URL & 回调接口。实现高并发。
4. 采用 Redis 记录图片上传状态，保证失败可以重试，提高系统的稳健性。
5. 采用混合检索的策略。利用 Elasticsearch 的 KNN 算法计算查询向量和文档向量及图片描述向量的余弦相似度，同时考虑关键词实际出现的频率与重要性，综合打分，提高检索的精准率与召回率。
6. 构建 RAG 检索流程：通过用户提问 + 检索片段拼接生成增强型 Prompt，在数据库中召回图片。
7. 采用基于 Redis 的对话历史管理机制，每个用户都有一个唯一的会话 ID，所有的对话内容都按照时间顺序存在 Redis 中，并设置了 7 天的过期时间，以便在多轮对话中保证上下文信息的完整性。
8. 项目采用了 Mockito 注解驱动的测试模式，践行测试驱动开发（TDD）的理念，每个业务功能都有对应的测试用例，包括正常流程和异常流程。
