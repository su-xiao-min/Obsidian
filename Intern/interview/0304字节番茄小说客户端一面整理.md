# 二、每类问题简要标准回答

我给你的是**面试可直接输出版逻辑结构**，不废话。

---

## ① 客户端 vs 后端区别

- 客户端：
  - 面向用户体验
  - 关注渲染性能、交互流畅度、生命周期管理
  - 单仓库多人协作，耦合度高
  - 需要强模块治理

- 后端：
  - 面向数据与业务逻辑
  - 微服务拆分，服务隔离
  - 分布式一致性、存储、扩展性
  - 强调系统可用性与扩展能力

---

## ② 安卓四大组件

- Activity（界面）
- Service（后台服务）
- BroadcastReceiver（广播）
- ContentProvider（数据共享）

---

## ③ Activity 生命周期（核心）

```
onCreate
onStart
onResume
onPause
onStop
onDestroy
```

设计目的：

- 解决资源释放
- 解决状态恢复
- 避免内存泄漏
- 适配前后台切换

---

## ④ 多态

- 编译期多态：重载
- 运行期多态：重写 + 父类引用指向子类对象
- 本质：动态绑定

---

## ⑤ 匿名内部类

- 没有类名的内部类
- 常用于一次性回调
- 编译后生成 Outer$1.class
- 持有外部类引用
- 可能导致内存泄漏

性能更高？
❌ 不成立。本质仍然是对象。

---

## ⑥ JVM GC 核心

- 可达性分析
- GC Roots：
  - 栈引用
  - 静态变量
  - 常量池
  - JNI

常见算法：

- 标记清除
- 复制
- 标记整理
- 分代收集

---

## ⑦ 内存泄漏场景（客户端常考）

- 静态持有 Activity
- Handler 非静态内部类
- 单例持有 Context
- 线程未关闭

---

## ⑧ 类锁 vs 对象锁

- 对象锁：锁实例
- 类锁：锁 Class 对象
- 字节码：
  - monitorenter / monitorexit

---

## ⑨ 虚拟地址转换

流程：

1. CPU 生成虚拟地址
2. MMU 查页表
3. 页表映射到物理页框
4. TLB 加速缓存

现代系统基本是分页为主。

---

## ⑩ 断点续传

HTTP Range 头：

```
Range: bytes=500-
```

服务器返回 206 Partial Content

---

## ⑪ Protobuf vs JSON

| 维度     | Protobuf | JSON |
| -------- | -------- | ---- |
| 格式     | 二进制   | 文本 |
| 体积     | 小       | 大   |
| 解析速度 | 快       | 慢   |
| 可读性   | 差       | 好   |

---

## ⑫ HTTPS 建立流程

1. Client Hello
2. Server Hello
3. 证书下发
4. 客户端验证证书
5. 生成对称密钥
6. 使用对称加密通信

---

- 架构治理难

> 客户端早期更混乱更难治理
> 后端上限更高
