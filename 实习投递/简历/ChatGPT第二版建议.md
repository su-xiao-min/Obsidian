## 二、逐段拆你现在这版的问题

### 1️⃣ 教育背景：没错，但可以更“值钱”

现在是标准安全写法，但**浪费了你的专业优势**。

你是**信息管理 + 数字文化**，这在做“图像检索 / 文献 / OCR / 搜索系统”时是**加分项**，但你一句都没用。

**问题：**

- 专业课程只是罗列，没有“和项目挂钩”
- 面试官看不出你是“做这个项目是顺的”，而不是硬拧的

**建议（不需要我现在帮你改，给你方向）：**

- 至少暗示一句：

  > 信息组织 / 信息检索 为后续图片索引与检索设计提供方法论基础
  > 不解释，点到为止。

------

### 2️⃣ 项目一：珞珈派的问题不是少，而是“功能向”

你现在对它的定位是“兜底项目”，这个判断**是对的**。
但你写法的问题在于：**它太像教学项目了**。

#### 问题点拆解

**① 描述层级太低**

- JWT + Redis 双令牌
- RabbitMQ 异步
- Redis 计数

这些是**“技术名词正确，但价值不突出”**的典型。

面试官的内心 OS 会是：

> 嗯，课程项目 or B站项目升级版。

#### ② 没有“为什么这样设计”

你写的是“我做了什么”，
但面试官想知道的是：

- 为什么不用单 Token
- 为什么用 MQ
- Redis 一致性方案的 trade-off

你现在一句都没给。

#### ③ 建议你给它的正确定位

这个项目应该承担的作用是：

> 证明你：
>
> - 会写**可维护的业务代码**
> - 对**鉴权、缓存、异步**不是停留在“能用”

所以你可以接受它**不炫，但要稳**。

------

### 3️⃣ 项目二：珞珈图谱是核心，但你在“自我降级”

这个项目**是你简历的命门**，也是你最值钱的部分。
但你现在写法有一个明显问题：

> 你在用“中性工程描述”，掩盖你其实在做的是一个**有设计难度的异步系统**。

#### 核心问题一：状态机写得太轻了

你现在写的是：

> 设计图片资源的多阶段状态流，区分持久化状态与过程状态，通过 Redis 管理索引流程

这是**非常正确的一句话**，
但它的杀伤力只有 30%。

**为什么？**

- 没说你解决了什么问题
- 没说“如果不这样会发生什么灾难”

面试官不会自动脑补你的聪明。

#### 核心问题二：OCR 回调这段容易被质疑

你写的是：

> 避免阻塞的时间消耗和轮询的消耗，实现高并发

面试官会直接追问：

- 回调失败怎么办
- OCR 返回慢怎么办
- 重复回调怎么办
- 服务重启怎么办

你简历里没有给“你考虑过”的任何痕迹。

#### 核心问题三：混合检索略显“概念堆叠”

KNN + 关键词是对的，但：

- 为什么不是纯向量
- BM25 在你系统里是不是兜底
- 哪些字段进向量，哪些不进

你写得像论文摘要，而不是工程设计。

------

## 三、你这版简历最需要补的一件事

不是新技术，而是：

> **“失败路径意识”**

现在你写的都是“我设计了什么”，
但真正区分初级和中级工程师的是：

- 哪一步可能失败
- 失败了系统会怎样
- 你如何保证它不会拖垮全链路

哪怕你只在珞珈图谱里**点出一条失败路径**，
面试官都会明显抬高对你的判断。

## 四、只基于你这份简历的【高命中率面试题】

下面这些问题，**90% 概率会被问**，而且是顺着你写的内容来的。

### 🔹 基础兜底（项目一）

1. 为什么要做 Access / Refresh 双 Token？
   如果 Redis 挂了会发生什么？
2. RabbitMQ 在你这个系统里解决的是什么问题？
   如果不用 MQ，后果是什么？
3. 你说“先写 MySQL 再删 Redis”，
   在并发场景下有没有脏数据窗口？

------

### 🔹 核心拷问（项目二，重点）

1. 你说“区分资源状态和过程状态”，
   **具体怎么区分？哪些进 MySQL，哪些只在 Redis？**
2. OCR 回调如果：
   - 回调丢失
   - 回调重复
   - OCR 成功但 ES 写入失败
     系统如何保证最终一致？
3. 为什么你选择 **MySQL 作为事实源，ES 作为视图**？
   如果 ES 数据全没了怎么办？
4. 你为什么不直接用“纯向量检索”？
   在什么情况下关键词匹配会比向量更靠谱？
5. 你的索引流程如果在 embedding 阶段失败，
   是整体回滚，还是局部补偿？

------

### 🔹 进阶加分（你如果答得好，面试官会明显认真）

1. Redis 状态机如果服务重启，
   如何避免“任务永久卡死”？
2. 你这个系统，**在哪一步最容易成为性能瓶颈？**
3. 如果图片量从 1 万涨到 100 万，
   你第一步会改哪里？
